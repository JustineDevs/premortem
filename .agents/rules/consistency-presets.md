# Consistency Presets

Canonical domain: writing, decision, UX, and engineering presets.

Parent: [CORE-BEHAVIOR.md](CORE-BEHAVIOR.md)

## Writing preset

```text
Tone: precise, spartan, technical, non-theatrical
Priority: clarity > inspectability > fluency
Forbidden: hype, fluff, motivational filler, generic assistant phrasing
Explanation: brief, direct, evidence-linked
```

Skills: `stop-slop`, `humanizer`, `cleanup`

## Decision preset

```text
Prefer narrower correct scope over broader uncertain scope.
Prefer explicit insufficiency over fabricated confidence.
Prefer reviewer control over autonomous publishing.
Prefer structured artifact generation over prose summaries.
```

Gated lanes: `align`, `flow`, `arch`, `vet`, `vibe`, `build`, `tdd`, `diagnose`, `cleanup`, `maestro`

## UX preset

```text
One primary action per step.
Truthful progress states only.
No hidden automation behind optimistic labels.
Review before publish.
Stable terminology everywhere.
```

## Engineering preset

```text
Schemas first.
Contracts before implementation.
Version all mutable artifacts.
Separate read models from publish actions.
Do not couple orchestration logic to UI copy.
```

Skills: `.agents/skills/engineering/` (mattpocock), `frontend/react-best-practices`, `frontend/next-best-practices`
